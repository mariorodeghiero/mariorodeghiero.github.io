SASS

"Melhorando seu CSS com SASS"

SASS é um pré-processador assim como o Less e o Stylus. Particularmente eu prefiro usar o SASS, então é sobre ele que vamos falar neste artigo. Mas porque usar um pré-processador? Neste artigo vamos conferir algumas das funcionalidades que o SASS oferece para resolvermos algumas questões que no CSS não é possível.

Variáveis

Um problema muito conhecido no mundo da programação é o DRY(Don't repeat yourself), as variáveis no Sass ajudam a resolver boa parte desta questão. Vamos ver como exemplo, o uso de cores no CSS: 

        

  .header {
    color: #000;
    background-color: #fff;
  }

  .links{
    color: #000;
  }

  .footer{
    background-color: #fff;
  }

Analisando este código, é possível notar que se repete o valor hexadecimal de algumas cores, agora imagine se alguém solicita uma alteração de background no "header" e "footer", você teria que alterar em dois locais diferentes e dependendo do projeto e solicitação, seria uma tarefa muito maçante. Com o SASS, podemos resolver esta questão rapidamente e de forma muito eficaz. 

        

  $defalt-color: #000;
  $bk-color: #fff;

  .header {
    ...
    color: $defalt-color;
    background-color: $bk-color;
    ...
  }

  .links{
    ...
    color: $defalt-color;
    ...
  }

  .footer{
    ...
    background-color: $bk-color;
    ...
  }

Mixins

As funcionalidades que os mixins oferecem são extremamente importante, pois com eles podemos definir estilos e reutiliza-los em nosso código. Dentro de um mixin pode conter muitas regras, mas o ideal é que ele não seja muito extenso, ele deve ser simples e funcional. 

        
  /**
   * Criando mixin
   */

  @mixin rounded-border($radius: 0.2em) {
    border-radius: $radius
  }

  /**
   * incluindo mixin com valor default
   */

  .button {
    ...
    @include rounded-border;
    ...
  }


  /**
   * Incluindo mixin com outro valor
   */

  .card {
    ...
    @include rounded-border(0.5em);
    ...
  }

Uma forma eficaz de criar um mixin é passando um valor padrão como argumento, assim você pode utilizar o mixin com valor padrão para um determinado componente e caso precise reutilizar o mixin em outro componente, basta informar o novo valor entre parentes como no código acima.

@extend

O @extend possui uma funcionalidade interessante, onde podemos criar uma classe ou ID com as regras de CSS desejada e para usar, basta usar "@extend + classe". Assim como o próprio nome diz, ele vai estender a classe ou ID com as regras CSS e quando gerado o arquivo CSS, no inicio do arquivo vai conter o nome do exetend acompanhado de todos locais onde ele foi chamado. 

        
  .shadow {
    -webkit-box-shadow: 0 2px 5px 0.3px rgba(0, 0, 0, 0.4);
    box-shadow: 0 2px 5px 0.3px rgba(0, 0, 0, 0.4);
  }

  .card {
    ...
    @extend .shadow
    ...
  }

%placeholder

O placeholder também é de grande utilidade e uma das diferenças entre "placeholders" e "mixins", é que no mixin podemos informar valores padrões porque ele trabalha com "@include". Já no placeholder utilizamos "@extend" onde não tem suporte para receber valores. Então se você precisar passar algum valor, o mixin é mais aconselhado, caso contrario você pode utilizar o placeholder da seguinte forma: 

        
  %shadow {
    -webkit-box-shadow: 0 2px 5px 0.3px rgba(0, 0, 0, 0.4);
    box-shadow: 0 2px 5px 0.3px rgba(0, 0, 0, 0.4);
  }

  .card {
    ...
    @extend %shadow
    ...
  }

O placeholder tem uma característica conhecida como "classe silenciosa", pois quando gerado o código CSS, o placeholder fica oculto, montrando somente onde ele foi inserido, o contrario do "@extend".

Conclusão

Essas são apenas algumas das inúmeras funcionalidades que o SASS disponibiliza para agilizar o processo de desenvolvimento, deixando nosso código mais performático. Vale a pena conferir a documentação diretamente no site do SASS e conhecer um pouco mais sobre o poder que ele oferece.
